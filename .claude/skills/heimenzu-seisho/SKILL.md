# 平面図 清書 ／ 割付図 作成スキル

株式会社アート・レイズの継続作業。現地の手書き図・島津等のレイアウト図から、提出用の平面図清書（SVG＋PNG）と、床・壁クロスの割付図を作成・更新する。本書が唯一の正本（手順書）。

初版のきっかけ：島津-リハビリ撮影-26（大分リハビリテーション病院 一般撮影室）。末尾に実例を残す。リポジトリ直下の `島津-リハビリ撮影-平面図_清書.svg` 等が完成例。新規案件は近い例を複製して寸法・開口・品番を差し替えると速い。

## 0. 成果物・命名・用紙サイズ

- 出力は **SVG**（正本）＋ **PNG**（共有用）。命名 `<案件キー短縮>-<種別>.svg/.png`。
  - 種別：`平面図_清書`／`床割付図`／`クロス割付図`／`内部仕上げ表`。
- **用紙サイズ（viewBoxの縦横比を用紙に合わせる）**：
  - **平面図清書**：A4縦（viewBox 例 1000×1414）または A4横（例 1120×792・1188×840）。
  - **内部仕上げ表**：A4横（例 1188×840）既定。室数・記載が多ければ A3横（同比で拡大）。
  - **割付図**：A4横／A3横（内容に応じて）。
  - A4横とA3横は同じ縦横比（√2≈1.414）。PNGも同比で出力（例 1568×1109＝A4横）。
- 同じ案件は**1組を上書き更新**し正本を1つに保つ（v2を増やさない）。
- 反映後は `data.json` の該当案件「備考」と `現地調査メモ_<案件キー>.md` も整合させる。
- **清書・割付図・仕上げ表の品番・寸法は必ず一致**させる（情報源＝同じ案件データ）。

## 1. 素材の入手（Google Drive）

- 写真は `案件 > <顧客> > <案件キー> > 写真`。**最新アップロードを必ず確認**（「最後に上げた画像」を取り違えない）。`search_files` を createdTime 降順で確認。
  - 例：`(parentId = '<写真ID>' or parentId = '<案件ID>') and mimeType contains 'image/'`
- `download_file_content` はトークン超過するので、保存された一時ファイルから base64 をデコードして読む。
- スマホ写真はEXIF回転に注意。`ImageOps.exif_transpose` か `rotate(-90/90, expand=True)` で正立 → 領域ごとに拡大判読。

## 2. 読み取りの大原則（手書き記号）

| 記号 | 意味 |
|---|---|
| **太いマジック（極太線）** | **壁**。清書はこの線を正として外形トレース |
| 細い鉛筆・ペン線 | 寸法・什器・補助線（壁ではない） |
| **壁（太線）の途切れ** | **建具（ドア）または窓**がある |
| 「柱」表記 | **壁柱**（壁面に付く＝室内へ突出する柱。独立柱ではない） |

- **面記号** `A=上 / B=右 / C=下 / D=左`（島津図「展開方向 A/B/C/D」と同じ。`zumen_app.html` も同定義）。
- 流れ方向の指定は必ず拾う。**流れは面ごとに違い得る**（本案件：床=縦／壁=縦／天井=横）。

## 3. 建具（ドア）の作図規約 ★最重要（`zumen_app.html` 準拠）

- **実線＝開いた扉そのもの**。吊元を芯に壁から **90°開いた位置**の扉板（長さ＝扉幅）。
- **点線＝戸先（扉先端）が描く弧の軌道**。
  - **吊元**＝「閉じた扉（壁と同化）」と「開いた実線扉」の**交点**。
  - 弧は **吊元中心・半径＝扉幅** の四半円。閉位置（壁上の戸先）→開位置（実線扉先端）を結び、**室内側へ膨らむ**。吊元と反対側を中心に逆向きへ膨らむのは誤り。
- 開口部に**閉じた扉（白四角）を描き、その上に開いた扉を重ねない**（閉と開の二重表現は誤り）。開口は壁を途切れさせるだけ。
- 開き勝手（内/外）・吊元（左/右）は現地図の弧に合わせる。
- **親子ドア**：親（幅広）・子（幅狭）。閉じると中央で合わさる。本案件は**親=右/子=左**。
- 標準寸法：片開きW900/H2100・親子W1200/H2100・両開きW1800・引違W1800・スライドW900。窓：中連窓/掃出/排煙/横滑り/縦滑り/嵌め殺し。
- **窓**：中連窓は **腰下（腰壁高）** と **窓高H** を区別（例：中連窓 W850×H850・腰下860）。「H」を腰下と取り違えない。

## 4. 壁柱の作図

- 壁面に付き**室内へ突出**。表現は柱記号に倣う：**塗り四角＋対角クロス(X)＋枠線**、「壁柱 ◯◯」。
- 壁沿いの長さ（例670）と室内突出（例459）を記録。

## 5. 品番（仕上げ）と 180°回転の対応 ★注意

島津図は**操作室=下**、現地手書きは**操作室=上**のことがあり、**180°回転**の関係。品番は島津図に記入されがちなので回転を考慮して各壁へ割当てる。

- **アンカー（隣接室名）で対応付ける**（上下左右でなく「操作室側/廊下側/受付側/空室側」）。180°では上下も左右も入替わる（点対称）。
- 各壁ラベルに「**面記号＋隣接室＋品番**」を併記。

## 6. 寸法線の表記規約 ★全図共通・必須

寸法は「室内の内寸法」を表す。

### 矢印の向き
- 矢印は端点を**外側から指す＝常に外向き**。
- 連続寸法（例 2241｜1280）の**境目は2本が向かい合う**（各区間が外向きなので共有点で先端が合わさる）。
- 実装：始点・終点で別マーカー（`dimS`/`dimE`）。単一 `dim`（両端同じ）は始点が内向きになり**不可**。
- ※流れ矢印（床流れ/クロス流れ）は方向指示なので本規約の対象外。

### 数字の位置・向き
- **横の寸法線**：数字は常に**線の「上」**（線より小さいy）。
- **縦の寸法線**：右側でも左側でも数字は常に**線の「左」**。
- **縦の数字の向き**：**左から読む**＝`transform="rotate(-90 …)"`。
- 合言葉：「**縦寸法は線の左、向きも左**」。

## 7. SVG作図スタイル・色

- viewBox例 `0 0 1120 760`。フォントは Noto Sans CJK JP 系。
- 線色：壁芯/壁柱 `#2c3e50`、床(長尺) `#cfe3f0`、建具 `#e8932e`、寸法 `#c0392b`、流れ `#2a6da3`、品番ラベル `#a8600a`。
- 構成：タイトル＋サブタイトル／室外形（壁=太線・開口で途切れ）／壁柱／建具（実線扉＋点線軌道）／窓／什器／寸法線／面記号A〜D／流れ矢印／凡例／仕上げ・数量メモ。
- **什器の色分け**（現地の色付け図に準拠。図中と凡例で一致＋文字ラベル併用）：
  - 照明40W×2＝黄 `#f6e6a8`／AC＝ピンク `#f3cfe1`／SP＝緑 `#c4e3c0`／換気扇＝青 `#bcd8f0`／非常灯＝紫 `#ead3ee`／床ピット＝緑ハッチ。
  - ※色付き手書き図は記号判別に有効（本案件で「黄帯＝照明、床ピットは別物」と誤読を訂正できた）。

### 再利用スニペット（defs：寸法マーカー・パターン）

```xml
<defs>
  <!-- 寸法矢印：外向き。始点dimS／終点dimE -->
  <marker id="dimS" markerWidth="10" markerHeight="10" refX="2" refY="5" orient="auto">
    <path d="M8,2 L2,5 L8,8" fill="none" stroke="#c0392b" stroke-width="1"/></marker>
  <marker id="dimE" markerWidth="10" markerHeight="10" refX="8" refY="5" orient="auto">
    <path d="M2,2 L8,5 L2,8" fill="none" stroke="#c0392b" stroke-width="1"/></marker>
  <!-- 流れ矢印（両端・方向指示） -->
  <marker id="ar" markerWidth="10" markerHeight="10" refX="5" refY="5" orient="auto">
    <path d="M1,1 L9,5 L1,9" fill="none" stroke="#2a6da3" stroke-width="1.4"/></marker>
  <!-- 床ピット=緑ハッチ／柱=グレーハッチ -->
  <pattern id="pitg" width="8" height="8" patternUnits="userSpaceOnUse" patternTransform="rotate(45)">
    <rect width="8" height="8" fill="#e2efe0"/><line x1="0" y1="0" x2="0" y2="8" stroke="#6fae6a" stroke-width="1.6"/></pattern>
  <pattern id="col" width="7" height="7" patternUnits="userSpaceOnUse" patternTransform="rotate(45)">
    <rect width="7" height="7" fill="#dfe7ec"/><line x1="0" y1="0" x2="0" y2="7" stroke="#9bb3c2" stroke-width="1.3"/></pattern>

```

## 8. 割付図への展開（清書寸法ベース。`材料割付_app_.html` 準拠）

- **床 長尺シート巾**：920/1320/1820/2000（既定1820）。流れ=縦→縦割り（W方向に列、各列長さ=L）。列数=ceil(W/巾)、縦継ぎ=列数−1。
- **壁クロス巾**：920（サンゲツFE標準）。開口控除。壁A〜D＝A:W×H／B:L×H／C:W×H／D:L×H。
- **貼り方向は面ごとに確認**（本案件：壁=縦貼り／天井=横流れ）。
  - 縦貼り：短冊=巾920×高さCH＋上下しろ。短冊数=ceil(壁長/920)。
  - 横流れ：920帯を長手方向に通す。帯数=ceil(短手/920)。
- 巾木は周長ベース（開口控除）。発注前に：FE品番のリピート（柄合わせ）有無、壁柱の見付・見込、入隅/出隅の回し、天井ルーバー開口控除を確認。

## 9. 内部仕上げ表（室別一覧表）

提出様式の一覧表。**用紙＝A4横（既定）／A3横**。清書・割付と**同じ品番・寸法**で作る。

- **列**：`室名｜床｜巾木｜壁｜天井｜その他部材・設備品`。各列見出しの下に小さく「仕上・品名・品番」。
- **行**：室ごとに1行。セルは複数行で「**仕上げ種別／品名／品番**」を記載（品番は太字）。
- **その他部材・設備品**：建具（種別＋W/H＋開き勝手）・窓（中連窓等＋腰下）・壁柱・CH・室内寸法など。
- **色の規約（クリップ準拠）**：
  - **赤字＝注記（要確認）** `#c0392b`。例「※展開図参照」「※現場判断」「※品番（高さでない）」「※リピート確認」。
  - **グレー字＝支給品（お見積り不要）** `#999`。
  - 流れ等の補足は青 `#2a6da3`。ヘッダ網 `#eef2f6`、罫線 `#888`。
- 末尾に **■備考** の大枠（元請・施工日程・数量・割付方針・注記）。
- 実例：`島津-リハビリ撮影-内部仕上げ表.svg/.png`（単室）。複数室はヘッダ下に行を増やす。

## 10. その他のリスト（内装仕上げ／照明器具／建具）

内部仕上げ表に加え、案件に応じて作成（A4横／A3横）。清書・仕上げ表と**品番・寸法を必ず一致**。色規約は共通（赤字=注記／グレー=支給／品番太字／流れ=青）。什器色も清書と共通（照明=黄・非常灯=紫 等）。

### (1) 内装仕上げリスト（面×室のスウォッチ表）
- 行＝面（天井／壁／床／巾木）、列＝室。各セル＝**スウォッチ＋品番**。壁が面別に複数品番なら4面分の小スウォッチを並べる。
- **品番判明品はメーカーサイト（サンゲツ等）で製品画像＋タテ/ヨコの柄リピートを取得**し、代表色スウォッチを**実画像に差し替え**＋リピート値を記載。
- **柄リピートはクロス割付の柄合わせに反映**（柄ありはタテリピート分の材料増＝歩留り低下）。パターン柄（例 FE-76751）は「柄合わせ要」を明示。
- ※サンゲツ公式・主要販売店は自動取得を **403で拒否**する場合あり。その時はかめさんが画像/リピートを支給、または手元ブラウザから取得して貼付。品番→カタログ(例 FINE 2023-26)・機能・巾(92cm)は検索で判明することが多い。
- 画像未取得時は代表色＋「画像/リピート＝要取得」を明記。実例：`島津-リハビリ撮影-内装仕上げリスト.svg/png`。

### (2) 照明器具リスト
- 様式A（新築・器具選定）：**場所別の列**に 符号・器具種別・メーカー・品番(型番)・数量・ランプ/W・色温(例2700K)・単価・**製品写真/姿図**・備考。復活/変更は**赤字＋日付**。
- 様式B（既存記録・改修）：**器具別の行テーブル**（符号／場所／種別／メーカー／品番／数量／ランプW／色温／備考）。既存・支給はグレー字。
- 符号：L=一般照明／E=非常・誘導。色は清書の什器色（照明=黄・非常灯=紫）と共通。
- 実例：`島津-リハビリ撮影-照明器具リスト.svg/png`（既存記録＋新築テンプレート列）。

### (3) 建具リスト（建具表）
- 建具ごとに：**符号（丸囲み D=ドア/W=窓）＋種類＋数量**、**姿図（W×H・▽FL・寸法）**、行＝箇所／仕上／見込／ガラス／付属金物、末尾に備考。
- 姿図：**開き勝手＝一点鎖線のくの字**（吊元側の高さセンターで折返し→戸先側へ開く）。**先（戸先）側にレバーハンドル `●—` を表示**。**親子は親のみハンドル・子扉はハンドル無**。窓は腰下高さを▽FLから明記。寸法は本書6章の規約に従う。
  - dash-dot＝`stroke-dasharray="7 2 2 2"`。くの字＝apex(吊元辺・H中央)から戸先辺の上下角へ2本。
- 開き勝手・吊元・寸法は清書/展開図に一致。既存建具は仕上/ガラス/金物を「既存（要確認）」。
- 実例：`島津-リハビリ撮影-建具リスト.svg/png`。

## 11. レンダリング・検証・コミット

```bash
pip install Pillow cairosvg            # 初回のみ
apt-get install -y fonts-noto-cjk      # 日本語フォント（PNG文字化け防止）
python3 -c "import xml.dom.minidom; xml.dom.minidom.parse('<f>.svg')"   # XML検証
python3 -c "import cairosvg; cairosvg.svg2png(url='<f>.svg', write_to='<f>.png', output_width=1568, output_height=1064)"
```
- 出力PNGを**必ず目視**。特に①ラベルの重なり ②ドア円弧の向き（吊元中心・室内へ膨らむ） ③壁の途切れ位置 ④寸法（矢印外向き・数字位置）を拡大確認。問題は該当ブロックのみ修正し再レンダリング。
- `data.json` はPythonで load→dump し整合性検証してから保存（壊れやすい）。差分は最小限。
- SVG/PNG/メモmd/data.json をまとめてコミットしPR更新。指定の開発ブランチへ push。
