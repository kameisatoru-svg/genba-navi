# Co-Work（Claude in Chrome）用プロンプト：内装仕上材データシート作成 → 案件フォルダ保存

現場ナビの「内装仕上げリスト／割付」を Claude Code 側で作るための**素材集め**を、Claude in Chrome（Co-Work）に依頼するためのプロンプト。
ブラウザ操作（メーカー公式の品番ページから**スウォッチ画像**と**リピート等の仕様**を取得）→ A4データシート(HTML→PDF) → **Google Drive 案件フォルダへ保存**まで行わせる。

> ★ **Co-Work側の正式プロンプト＝`finish-material-datasheet`**（かめさんが作成し Claude in Chrome 側に保存）。
> 実運用はこれを実行し、**品番リストと保存先だけ差し替える**。本ファイルは同じ意図のスペック（同期用の控え）。
> Claude Code 側から Co-Work 作業を案内するときは、**長文を貼らず**「`finish-material-datasheet` を実行（品番・保存先を差替え）」と案内する。
> ※`finish-material-datasheet` の中身を更新したら、その内容を共有してもらい本ファイルも同期（drift防止）。

---

## 使い方
下の「プロンプト本文」をコピーして Co-Work に貼り、`{{ }}` を案件に合わせて埋める。

## ポイント（なぜこの形か）
- 仕様の数値（リピート等）は**テキストとして**配置させる（スクショ画像にしない）。→ Claude Code が `pymupdf` で**画像＝スウォッチ／テキスト＝数値**を自動抽出できる。
- 画像は**メーカー公式のスウォッチ**（色柄が分かる生地正面。部屋の施工イメージ写真ではない）。
- **ファイルは10MB未満**を目安に（Drive取り込みの都合）。多ければ分割。
- 保存先は**案件フォルダ**。ファイル名を固定形にして Claude Code 側が拾いやすくする。
- **床材・巾木も含める**（リピートが無くても画像・品番・寸法は取得）。

---

## プロンプト本文（ここからコピー）

```
あなたはブラウザを操作するアシスタントです。内装仕上材の「製品スウォッチ画像」と「仕様（リピート等）」をメーカー公式サイトから集め、A4データシート(PDF)にまとめ、Google Driveの案件フォルダに保存してください。

# 入力
- 案件キー: {{例: 島津-リハビリ撮影-26}}
- Drive保存先フォルダ: {{案件 > 顧客 > 案件キー（または写真/書類サブフォルダ）}}
- 品番リスト（部位 / メーカー / 品番）:
  - 壁・天井: サンゲツ / {{FE-76751, FE-76251, FE-76728, FE-76109, FE-76202}}
  - 床: {{メーカー不明なら品番で特定}} / {{PT-105M}}
  - 巾木: {{田島(タジマ)}} / {{No.80}}

# 手順
1. 各品番について、メーカー公式の商品詳細ページを開く（サンゲツは公式で品番検索→詳細。床・巾木は各メーカー公式。メーカー不明な品番は品番で検索して特定する）。
2. 各品番について次を取得する:
   - 品番 / 商品名・シリーズ / 掲載カタログ（例 FINE 2023-26）
   - 巾(cm)
   - タテリピート / ヨコリピート(cm)（無ければ「リピートなし」と明記。床・巾木でリピート概念が無ければ寸法/巾を記載）
   - 柄合わせ種別（●無地貼可 / ステップ柄合せ / 柄合せ / リピートなし のいずれか）
   - 機能（防かび・抗菌・表面強化 等）・標準価格・防火種別
   - 製品スウォッチ画像（色柄が分かる生地の正面画像。できるだけ高解像度。部屋の施工イメージ写真は不可）
3. 上記を 1品番=1ブロック で A4横のHTMLにまとめる（2×2など複数ブロック可）。各ブロックに「品番(テキスト) + スウォッチ画像 + 仕様テキスト(リピート含む)」を配置する。
   ※重要: 仕様の数値は必ず「文字（テキスト）」で配置する。スクリーンショット画像にしないこと。画像はスウォッチのみ。
4. そのHTMLをPDFに保存する（印刷→PDF、または直接PDF出力）。ファイルサイズは10MB未満を目安に（超える場合は品番を分けて複数PDFにする）。
   - ファイル名: 仕上材_{{案件キー}}_品番データ.pdf （分割時は _1, _2 を付与）
5. そのPDFを Google Drive の保存先フォルダ「{{Drive保存先フォルダ}}」にアップロードする。
6. 完了したら、保存したファイル名・保存先・取得できた品番／できなかった品番（理由）を報告する。

# 注意
- 画像は色再現が正確な公式画像のみ使用（第三者サイトの画像は避ける）。
- 取得できない品番は「未取得（理由）」と明記し、他を止めない。
- 床材・巾木も必ず含める。
```

## 受け取り後（Claude Code 側＝私の処理）
- Drive 案件フォルダ or アップロードされた `仕上材_<案件キー>_品番データ.pdf` を取り込み、`pymupdf` で各品番のスウォッチ画像＋リピートを抽出。
- 内装仕上げリストに**実画像を貼付**、リピートを**クロス割付の柄合わせ（要否・倍数切上げ）**へ反映。
- 「仕上げ表／内装仕上げリスト／割付」を同じ品番・寸法で更新。
