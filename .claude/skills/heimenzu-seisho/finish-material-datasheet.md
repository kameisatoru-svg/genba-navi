# 仕上材データシート作成スキル（finish-material-datasheet）v1

> ★ このファイルは **Co-Work（Claude in Chrome）側に保存された正式スキル `finish-material-datasheet` の同期コピー**（正本＝Co-Work側）。
> かめさんが正本を更新したら本文を共有してもらい、本ファイルを同期する（drift防止）。
> Claude Code 側から素材集めを案内するときは長文を貼らず「`finish-material-datasheet` を実行（品番・保存先を差替え）」と案内する。
> 受領した `仕上材_<案件キー>_品番データ.pdf` は Claude Code が `pymupdf` で取り込み、内装仕上げリスト／クロス割付（柄合わせ）へ反映する。

## 概要

部位／メーカー／品番のリストを入力に、各品番のメーカー公式ページから
**(1) 製品スウォッチ画像（色柄が分かる生地正面・高解像度。施工イメージ写真は不可）**と
**(2) 仕様テキスト（巾・リピート・柄合わせ・機能・標準価格・防火種別・材質など）**を収集し、
**A4縦・1品番＝1ブロック（フル幅）** のデータシートPDFを生成して、
Google Drive の **案件＞顧客＞案件キー** フォルダに保存する。

設計の肝：
- **仕様の数値は必ずテキスト（文字）で配置。スウォッチ以外の画像（スクショ等）は使わない。**
- スウォッチは **生地正面のみ**。部屋の施工イメージ／パース画像は使用しない。
- レイアウトは **A4縦・横幅は1品番フルに使い、縦に流す**。分量が多ければ自動で2枚目以降へ。

---

## 入力

- **案件キー**（例: `島津-リハビリ撮影-26`）— PDFファイル名・保存先フォルダ名に使う
- **Drive保存先**（既定: 案件＞顧客＞案件キー の標準階層。案件キーでDrive内検索して特定）
- **品番リスト**（部位 / メーカー / 品番）。例:
  - 壁・天井: サンゲツ / FE-76751, FE-76251, …
  - 床: タジマ / PT-105M
  - 巾木: タジマ / No.80

不足があれば AskUserQuestion で案件キー・保存先を確定してから着手する。

---

## 手順

### Phase 1. 各品番の公式ページを特定し、スウォッチURL＋仕様を取得

ブラウザ操作は **Claude in Chrome（`mcp__Claude_in_Chrome__*`）** を使う。
`get_page_text` で仕様テキスト、`javascript_tool` で画像URL抽出が速い。
**取得できない品番は「未取得（理由）」と明記し、他を止めない。**

#### ▼ サンゲツ（検証済み）
- 公式商品ページURL: `https://www.sangetsu.co.jp/product/detail/{品番}/`
  - **ハイフンなし**（`FE-76751` → `FE76751`）。`/search/?q=` は無効。
- 仕様は `get_page_text` で取得できる。代表項目：
  掲載カタログ（例「23-26 ファイン（FINE）」）／標準価格（例「1,000円/M , 1,090円/㎡（税別）」）／
  規格（例「92cm ×50m乱」→ 巾）／リピート（例「タテ64.2cm ヨコ92cm」）／
  リピート備考（例「ステップ柄合せ」）／防火種別（例「2-3」）／機能。
- **スウォッチ画像は「chip」を使う（生地正面）**。各品番ページの画像は3種類あり、
  用途を取り違えないこと：
  - `…/img/qrimage/…Q01.jpg` → **部屋の施工イメージ（使用不可）**
  - `…/img/repeat/…R01.jpg` → リピート図（使用しない）
  - `…/img/chip/…C01.jpg` → **生地正面チップ＝これを使う**
  - 取得法（商品ページ上で実行）:
    ```js
    Array.from(document.querySelectorAll('img')).map(i=>i.src)
      .filter(s=>/\/img\/chip\//.test(s))
    ```
- リピート無し（石目・塗装調など）は公式ページにリピート行が出ず → **柄合わせ＝「●無地貼可」** と判定。
  リピートありは「柄合せ」、リピート備考に「ステップ柄合せ」とあればそれを採用。
- 注意: 一部品番（例 FE-76251）は **規格（巾）が公式ページに数値表示されない**。
  その場合は捏造せず「公式商品ページに数値表示なし（見本帳参照）」と記載する。

#### ▼ 田島ルーフィング（タジマ）床材・巾木（検証済み）
- 旧 `tajima.jp/flooring/...` は**ソフト404／トップへリダイレクト**。新サイト
  **`https://flooring.tajima.jp`** を使う（製品は数値ID `/product/{id}`）。
- 品番→IDの特定は **カテゴリ一覧から絞り込む**のが確実：
  - カテゴリ・シリーズ一覧: `https://flooring.tajima.jp/search-by-category-and-series`
  - 主要カテゴリID（検証済み）: **ベーシックシート＝`/category/11`**（パーマリューム系・PT-xxxM等を含む）、
    **巾木＝`/category/33`**（ソフト巾木 HB-xx・長尺巾木 LB-xx 等）。
  - カテゴリページで「もっと見る」を押し切ってから、全 `a[href*="/product/"]` を集めて品番文字列で grep。
    ```js
    // 例: PT-105M を探す
    Array.from(document.querySelectorAll('a'))
      .map(a=>({t:a.innerText.replace(/\s+/g,' ').trim(), h:a.href}))
      .filter(x=>/\/product\/\d+/.test(x.h) && /PT-?105/i.test(x.t))
    ```
- 商品ページで `get_page_text` から基本情報を取得：
  製品名／厚さ／価格A（円/㎡）・価格B（円/m）／材質区分（例「複層ビニル床シート FS」「軟質ビニル巾木」）／
  寸法（例「2.0mm x 1820mm x 9M」）／重量／梱包／機能（例 ノーワックス・抗菌・抗ウイルス・重歩行・床暖対応）。
- **スウォッチ画像は `alt="product-top-image"`（1000×1000）を使う＝生地正面**。
  - `product-image-2`（2000px等）は**部屋の施工イメージ（使用不可）**。`tlab.style/...` もパース画像。
  - 取得法（商品ページ上）:
    ```js
    Array.from(document.querySelectorAll('img'))
      .filter(i=>i.alt==='product-top-image').map(i=>i.currentSrc||i.src)
    ```
- **巾木の品番「No.XX」はカラー番号**。コード体系は `HB-{No}-{高さ}`（ソフト巾木、高さ 40/60/75/100mm 展開）。
  高さ指定が無ければ標準的な **60mm（HB-{No}-60）** を代表に取得し、仕様に「高さ 40/60/75/100mm 展開」と注記。
- **床材・巾木はリピート概念が無い** → リピート欄は「…につきリピートなし（寸法・巾を記載）」、柄合わせは「—」。

#### ▼ それ以外のメーカー（汎用）
1. `WebSearch` で「{メーカー} {品番} 仕様」「{品番} 商品ページ」等を検索し、**メーカー公式**の商品詳細ページを特定
   （第三者の通販サイト画像は使わない）。メーカー不明な品番は品番だけで検索して特定する。
2. 公式ページから上記と同じ項目を取得。**スウォッチは必ず生地正面（フラットな色柄）**。
   施工イメージ・パース・room scene のサムネは避ける。判別が怪しい画像は
   ブラウザで画像URLへ移動し `screenshot` で目視確認してから採用する。
3. リピート概念が無い建材（床・巾木・タイル等）は寸法・巾を記載。

### Phase 2. スウォッチ画像をサンドボックスへダウンロード
- 作業フォルダ直下に `sw/` を作り、各品番の画像を `curl` で取得（CDNはサンドボックスから到達可）。
  ファイル名は品番ベース（例 `FE76751.jpg`, `PT-105M.jpg`, `HB-80.jpg`）。
  ```bash
  curl -s -o sw/FE76751.jpg "https://contents.sangetsu.co.jp/img/chip/FE76751_..C01.jpg"
  ```
- メーカー公式CDNからの`fetch`はCORSで失敗することがある。その場合は
  **画像URLへ直接 navigate してから同一オリジンで取得**するか、サンドボックス`curl`を使う。

### Phase 3. items.json を作って A4縦データシートPDFを生成
- `scripts/build_datasheet.py` を使う。`items.json`（配列。1要素=1品番）を作り、`sw/` に画像を置いて実行：
  ```bash
  pip install weasyprint --break-system-packages -q   # 初回のみ
  python3 scripts/build_datasheet.py items.json "案件キー" "仕上材_案件キー_品番データ.pdf"
  ```
- items.json の各要素キー: `part, maker, code, img, name, cat, haba, rep, gara, func, price, fire`
  （詳細は build_datasheet.py 冒頭のドキュメント参照）。
- レイアウト（スクリプトに実装済み）:
  - **A4縦**・余白広め。1品番＝フル幅カード。ヘッダ帯に品番（大）＋部位／メーカー、
    本文は左にスウォッチ（46mm角）・右に仕様テーブル（ラベル＋値、値は折返しで全文表示）。
  - 部位で色分け（壁・天井=青 / 床=緑 / 巾木=茶）。末尾に柄合わせ凡例＋備考。
  - カードは `page-break-inside:avoid` で、分量が多ければ自動で次ページへ流れる。
- 日本語は **Noto Sans CJK JP**（サンドボックスに既存）で描画される。

### Phase 4. PDFを検証
- `pdftoppm -r 110 -png 出力.pdf page` で各ページを画像化し、`Read` で目視確認：
  **文字切れが無いか／スウォッチが生地正面か／仕様が全項目テキストで出ているか**。
- サイズは **10MB未満目安**（通常1MB前後。超える場合のみ画像を縮小 or `_1,_2` で分割）。
- ファイル名: `仕上材_{案件キー}_品番データ.pdf`（分割時 `_1, _2`）。

### Phase 5. Google Drive の案件フォルダへアップロード
- 案件キーでフォルダ検索 → 親が「顧客名」フォルダ（さらに親が案件トップ）であることを確認:
  ```
  search_files: title contains '{案件キー}' and mimeType = 'application/vnd.google-apps.folder'
  ```
- アップロードは **Claude in Chrome** で Drive Web から行う（Drive MCP の create_file は
  base64インライン必須で大きいPDFには不向き）。手順:
  1. フォルダURL `https://drive.google.com/drive/folders/{id}` へ navigate。
  2. 「新規」→「ファイルのアップロード」をクリック（隠し `input[type=file]` が生成される。
     ネイティブダイアログは無視してよい）。
  3. `find` で file input の ref を取得 → `file_upload` で **outlookのWindowsパス**（outputsフォルダ）を指定。
  4. 同名が既存なら「アップロード オプション」ダイアログ →
     **「既存のファイルを置き換える」**（＝新バージョンとして更新。版履歴は残る）→「アップロード」。
  5. 「N件のアップロードが完了しました」を screenshot で確認。
- 削除・共有権限変更はしない。置き換え（版更新）は本人の自ファイル更新依頼の範囲でのみ行う。

### Phase 6. 完了報告
- 報告する内容: **保存ファイル名／保存先（案件＞顧客＞案件キー）／取得できた品番・できなかった品番（理由）**。
- present_files でローカルPDFも提示。Sources に使用した公式ページURLを列挙。

## 落とし穴メモ
- サンゲツ `qrimage` ／ タジマ `product-image-2` は **施工イメージ**。スウォッチに使わない。必ず生地正面（chip / product-top-image）。
- 旧 `tajima.jp/flooring/...` はリダイレクト＆ソフト404（HTTP200でも「お探しのページが見つかりません」本文）。新 `flooring.tajima.jp` を使う。
- メーカー公式CDNへの `fetch` はCORSで弾かれることがある → 画像URLへ navigate して同一オリジン取得 or サンドボックス `curl`。
- 公式に数値が載っていない仕様（例 サンゲツ一部品番の規格＝巾）は **捏造せず「公式未掲載（見本帳参照）」** と明記。
- 巾木「No.XX」はカラー番号であって高さではない。高さは別途（既定60mm・40/60/75/100mm展開）。

## 環境メモ
- PDF生成: `weasyprint`（A4縦・CSS @page）。日本語フォント Noto Sans CJK JP はサンドボックスに既存。
- 画像CDN（contents.sangetsu.co.jp / d1hp9r2rl3unti.cloudfront.net 等）はサンドボックス `curl` で到達可。
- 検証画像化: `pdftoppm`（poppler-utils）。

## Claude Code 側の受け取り処理（参考）
- 受領 PDF（`仕上材_<案件キー>_品番データ.pdf`、10MB未満）を Drive から取り込み、`pymupdf(fitz)` で
  各品番のスウォッチ画像（埋め込み image）と仕様テキスト（リピート等）を抽出。
- 内装仕上げリストに実画像を貼付、リピートを**クロス割付の柄合わせ**（無地貼可/リピート無＝不要、パターン柄/ステップ柄合せ＝要・短冊長をタテリピート倍数に切上げ）へ反映。
