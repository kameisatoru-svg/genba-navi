# 平面図 清書 作業手順書（現地手書き → SVG清書）

株式会社アート・レイズ／継続作業の標準手順。現地の手書き図・島津等のレイアウト図から、提出用の平面図清書（SVG＋PNG）を作成する際は本書に従う。
初版作成のきっかけ：島津-リハビリ撮影-26（大分リハビリテーション病院 一般撮影室）。本書末尾に実例を残す。

---

## 0. 成果物と命名

- **SVG**（正本）と **PNG**（共有用）の2点を出力。ファイル名は `<案件キーの短縮>-平面図_清書.svg / .png`。
  - 例：`島津-リハビリ撮影-平面図_清書.svg` / `.png`
- 同じ案件の清書は**1組を上書き更新**して正本を1つに保つ（v2等を増やさない）。
- 反映後は `data.json` の該当案件「備考」と `現地調査メモ_<案件キー>.md` も同内容に整合させる。

## 1. 素材の入手（Google Drive）

- 案件写真は Drive：`案件 > <顧客> > <案件キー> > 写真`。最新アップロードを必ず確認する。
  - **「最後に上げた画像」を取り違えない**こと。`search_files` を createdTime 降順で確認し、当日アップ分を特定する。
  - 検索例：`(parentId = '<写真フォルダID>' or parentId = '<案件フォルダID>') and mimeType contains 'image/'`
- 画像は `download_file_content` がトークン超過するため、保存された一時ファイルから base64 をデコードして読む。
- スマホ写真は EXIF で回転していることが多い。`ImageOps.exif_transpose` または `rotate(-90/90, expand=True)` で正立させ、領域ごとに拡大して判読する。

## 2. 読み取りの大原則（手書き図の記号）

| 記号 | 意味 |
|---|---|
| **太いマジック（極太線）** | **壁**（清書ではこの線を正として外形をトレース） |
| 細い鉛筆・ペン線 | 寸法・什器・補助線（壁ではない） |
| **壁（太線）の途切れ** | **建具（ドア）または窓**が存在する |
| 「柱」表記 | **壁柱**（独立柱ではない。壁面に付く＝室内へ突出する柱） |

- **面記号**：`A=上 / B=右 / C=下 / D=左`（島津図の「展開方向 A/B/C/D」と同じ。`zumen_app.html` の柱・建具ロジックも同じ定義）。
- 流れ方向の指定があれば必ず拾う（例：**床流れ＝縦／クロス流れ＝横**）。

## 3. 建具（ドア）の作図規約 ★最重要

`zumen_app.html`（図面アプリ）の定義に合わせる。間違えやすいので厳守。

- **実線（ゴールド）＝開いた扉そのもの**。吊元を芯に壁から **90°開いた位置**の扉板を描く（長さ＝扉幅）。
- **点線＝戸先（扉の先端）が描く弧の軌道**。
  - **吊元**＝「閉じた扉（壁と同化）」と「開いた実線扉」の**交点**。
  - 弧は **吊元を中心・半径＝扉幅** の**四半円**。閉位置（壁上の戸先）→開位置（実線扉の先端）を結ぶ。
  - **室内側へ膨らむ**向き。吊元と反対側を中心にして逆向きに膨らむのは誤り。
- 開口部に**閉じた扉（白い四角）を描いて、その上に開いた扉を重ねない**（閉と開の二重表現は誤り）。開口は壁を途切れさせるだけ。
- 開き勝手（内開き/外開き）・吊元（左/右）は現地図の弧の向きに合わせる。本現場は全て内開き。
- **親子ドア**：親（幅広）と子（幅狭）。本現場は **親＝右／子＝左**。両扉とも内開きで、閉じると中央で合わさる。
- 建具種別と標準寸法（アプリ準拠）：ドア(片開き)W900/H2100・親子ドアW1200/H2100・両開きW1800・引き違いW1800・スライドW900。窓：中連窓・掃出窓・排煙窓・横滑り・縦滑り・嵌め殺し(FIX)。

### 窓
- 中連窓は **腰下（腰壁高）** と **窓高(H)** を区別して記録（例：中連窓 W850×H850・腰下860）。「H」を腰下と取り違えない。

## 4. 壁柱（かべばしら）の作図

- B面等の壁面に付く柱。**室内へ突出**する。
- 表現は図面アプリの柱記号に倣う：**塗りつぶし四角＋対角クロス（X）＋枠線**、「柱 ◯◯」表記。
- 寸法は壁沿いの長さ（例：670）と室内への突出（例：459）を記録。

## 5. 品番（仕上げ）と 180°回転の対応 ★注意

島津のレイアウト図は **操作室＝下**、現地手書き図は **操作室＝上** のことがあり、**両者は180°回転**の関係になる。品番（クロス等）は島津図に記入されることが多いので、**回転を考慮して各壁へ対応付ける**。

- **アンカー（隣接室名）で対応付ける**のが安全（上下左右ではなく「操作室側／廊下側／受付側／空室側」で紐付け）。
- 180°回転では **上下も左右も入れ替わる**（点対称）。
- 各壁ラベルには「**面記号＋隣接室＋品番**」を併記し、取り違いを防ぐ。

## 6. SVG作図のスタイル指針

- viewBox 例 `0 0 1120 760`。フォントは Noto Sans CJK JP 系を指定。
- 色：壁芯/壁柱=#2c3e50、床(長尺)=#cfe3f0、建具=#e8932e、寸法=#c0392b、流れ=#2a6da3、品番ラベル=#a8600a。
- 構成：タイトル＋サブタイトル／室外形（壁＝太線・開口で途切れ）／壁柱／建具（実線扉＋点線軌道）／窓／什器（床ピット・AC・SP・非常灯・換気扇等は概略）／寸法線／面記号A〜D／流れ矢印／凡例／仕上げ・数量メモ。
- 什器・凡例の記号は手書き図の凡例に合わせる：照明40W×2／AC=エアコン／SP=スピーカー／換気扇／非=非常灯／床ピット。

## 7. レンダリングと検証

```bash
pip install Pillow cairosvg      # 初回のみ
apt-get install -y fonts-noto-cjk   # 日本語フォント（PNGの文字化け防止）
python3 -c "import xml.dom.minidom; xml.dom.minidom.parse('<file>.svg')"   # XML検証
python3 -c "import cairosvg; cairosvg.svg2png(url='<file>.svg', write_to='<file>.png', output_width=1568, output_height=1064)"
```

- 出力PNGを**必ず目視**。特に①ラベルの重なり ②建具の円弧の向き（吊元中心・室内へ膨らむ） ③壁の途切れ位置 を拡大して確認する。
- 問題があれば該当ブロックのみ修正し再レンダリング。
