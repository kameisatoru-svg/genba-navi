# 工具台帳登録スキル

工具・機材・車両を「工具・機材台帳」に登録する手順を定めたスキル。
**商品を登録するときは、本体情報だけでなく「取扱説明書」と「商品画像」を必ず一緒に保管する。**

## 対象データと保存先

| 種別 | 保存先 | data.json 上の記録 |
|---|---|---|
| 工具レコード本体 | `data.json` の `"工具"`（配列） | 下記フィールド一式 |
| 商品画像・全体写真 | `tool_photos/` に個別コミット | `写真`（パス文字列の配列） |
| 銘板・型番ラベル写真 | `tool_photos/` に個別コミット | `銘板写真`（パス文字列の配列・型番証跡） |
| **取扱説明書（PDF/画像）** | **`tool_manuals/` に個別コミット** | **`取扱説明書`（`[{名前, パス}]` の配列）** |

リポジトリ：`kameisatoru-svg/genba-navi`（GitHub Pages 公開）。
データは GitHub 上の `data.json` が正本。ローカル編集なら編集後に push、API 経由なら Contents API で読み書きする。

### 工具レコードの全フィールド

```
id / 名称 / カテゴリ / メーカー / 型番 / 保有数 / 状態 / 保管場所 /
取得日 / 取得価格 / 仕入先 / 使用方法 / 応用 / 注意事項 /
メンテ履歴(配列) / 次回点検 / 写真(tool_photos/配下のパス配列) /
銘板写真(tool_photos/配下のパス配列) / 取扱説明書([{名前, パス}]の配列) /
備考 / 原価ID / 登録日 / 更新日
```

- `id`：新規は `T-<base36タイムスタンプ>-<乱数4桁>` 形式（例 `T-mprzrywd-3rg9`）。
- `カテゴリ`：電動工具 / 手工具 / 測定器 / 機械・発電機 / 車両 / 収納・運搬 / 撮影・IT / 消耗品・付属 / その他 のいずれか。
- `登録日`は初回のみ。更新時は `更新日` を当日に更新し `登録日` は維持する。

## 登録手順

### Step 1. 商品情報を集める
- 写真・型番・カタログ・ユーザー入力から、名称／メーカー／型番／カテゴリ／使用方法／応用／注意事項を確定する。
- 写真があれば AI（vision）で自動推定してよい（工具台帳_アートレイズ.html の「写真からAIで自動入力」と同じ方針）。
  銘板/型番ラベルの接写があれば、そこに印字された メーカー／型番／定格 を最優先で読み取る。

### Step 2. 商品画像を保管する（必須・あれば）
- 商品の全体写真やメーカー公式・カタログの製品画像があれば、`tool_photos/` に保存する。
- ファイル名：`<id>_<base36タイムスタンプ><連番>.jpg`（銘板は `<id>_plate_<...>.jpg`）。
- 画像は長辺1280px前後・JPEG品質0.72程度に圧縮してから保存（銘板は1500px・0.85）。
- 保存したパスを `写真`（または銘板なら `銘板写真`）配列に追加する。
- **画像が1枚も無い場合のみ省略可。** 入手できる画像があるのに添付しないことは避ける。

### Step 3. 取扱説明書を保管する（必須・あれば）
- メーカー公式サイト等から、その型番の **取扱説明書PDF**（無ければ取説画像）を入手する。
  - 型番が分かっているときは「<メーカー> <型番> 取扱説明書 PDF」で検索して公式PDFを優先。
  - 入手できた場合は内容が当該型番のものか（型番・機種名）を確認する。
- `tool_manuals/` に保存する。ファイル名：`<id>_<base36タイムスタンプ><連番>.<拡張子>`（PDFは `.pdf`）。
- PDFは圧縮せずそのまま保存。取説が画像のときは長辺1800px・JPEG0.85程度に。
- `取扱説明書` 配列に `{"名前":"元のファイル名 or 機種名 取扱説明書", "パス":"tool_manuals/<...>"}` を追加する。
- **取説がどうしても見つからない場合のみ** `取扱説明書` は空配列 `[]` とし、`備考` に「取説未入手」と記す。

### Step 4. レコードを書き込む
- 既存 `id` があれば更新（`登録日`維持・`更新日`を当日に）、無ければ新規追加。
- `data.json` の `"工具"` 配列へ反映し、`最終更新` を当日（YYYY-MM-DD）に更新。
- コミットメッセージ例：`工具登録: <名称>` / `工具更新: <名称>` / `工具 取扱説明書: <名称>` / `工具写真追加: <名称>`。
- 画像・取説の個別ファイルと `data.json` は別コミットで構わない（アプリ実装と同じ）。

### Step 5. 確認
- 登録後、`写真` と `取扱説明書` が実在パスを指しているか（`tool_photos/` `tool_manuals/` 配下に存在するか）を確認する。
- 詳細画面（工具台帳_アートレイズ.html）で取説リンクが開けること、商品画像が表示されることを想定して整合をとる。

## 注意
- 取説PDFは著作権物。社内の台帳保管目的に限定し、再配布はしない。
- 大きすぎるPDF（数十MB超）は GitHub Contents API で失敗することがある。その場合は分割や軽量版を検討する。
- 写真・取説のパスは必ず `tool_photos/` `tool_manuals/` からの相対パスで記録する（先頭に `./` は付けない）。
