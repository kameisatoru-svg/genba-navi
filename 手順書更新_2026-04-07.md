## ■ 絶対ルール（全アプリ共通）

### 作業ベースの原則
- `/mnt/project/`は読取専用・古いバージョンの可能性あり・**絶対に作業ベースにしない**
- 作業ベースは必ず`/mnt/user-data/uploads/`（かめさんアップロード）またはGitHubから`web_fetch`で取得したもの
- **GitHubが常に正の状態**

### デプロイフロー

#### 方法A：auto_push.bat（推奨）
1. 作業開始時に `auto_push.bat` を起動
2. 私がファイル出力（present_files）
3. かめさんがダウンロード → genba-naviフォルダに保存
4. auto_pushが自動検知して git pull → add → commit → push
5. 私が30秒待ってからweb_fetchで内容確認
6. 確認OKを報告してから次の作業へ

#### 方法B：push.bat（従来方式）
1. 私がファイル出力（present_files）
2. かめさんがダウンロード → 上書き → push.bat実行 → 「プッシュしました」と報告
3. 私が30秒待ってからweb_fetchで内容確認

## ■ zumen_app.html（図面アプリ）

### 現状（ver.56以降）
- 平面図：開口部（ドア・窓）・設備・間仕切り・柱を配置可能
- id採番：全面通しのグローバル連番（面間重複防止）
- 展開図：開口部・設備ドロワーはphase分離（erase→内法白塗り→draw）

### 設計ルール（絶対遵守）
1. 外周寸法線の端点は内壁面（`pox, poy, pox+iW*sc, poy+iD*sc`）
2. 寸法値は`iW/iD`（内寸）を表示
3. 四角モードは`pox=ox`・`iW=W`、多角形モードは`pox=ox+Tpx/2`・`iW=_innerW`

## ■ GLM BLE連携（完成済み）

| 項目 | 内容 |
|------|------|
| 機器 | Bosch GLM 50-27CG（JP版） |
| 起動コマンド | 接続後に `C0 55 02 01 00 1A` をCharA1にWRITE |
| パケット構造 | bytes[7-10]=IEEE-754 float LE（メートル）→mm変換 |
| UUID（CHAR_A1） | `02a6c0d1-0451-4000-b000-fb3210111989` |
| 注意 | MeasureOnApp（Bosch公式）は接続前に完全終了必要 |
