## 🎨 ダッシュボード機能要件

### Section 1: ヘッダー
- タイトル「🏗️ ART-RAYS ダッシュボード」
- 最終更新日表示(data.jsonの"最終更新"から)
- 「更新」ボタン(data.json再fetch)
- 「印刷」ボタン
- 現在日時表示

### Section 2: KPIカード(4枚横並び)
| カード | 計算内容 |
|---|---|
| 進行中案件数 | ステータスが「中止」「入金済み」以外の件数 |
| 今月の見積総額 | 当月発行の見積金額合計 |
| 未入金合計 | 入金状況=minyukinの請求金額合計 |
| 当月施工予定 | 当月内の施工予定案件数 |

### Section 3: ステータスカンバン(横スクロール可)
6列構成:
1. 🟦 見積提出済み
2. 🟨 施工予定
3. 🟧 施工中
4. 🟩 請求済み
5. 💰 入金済み
6. ✅ 完了

各列に該当案件のカードを縦に並べる。カード内容:
- 案件キー(太字)
- 正式工事名
- 顧客略
- 最新金額(見積>請求のうち最新)
- 担当ステータス補足(備考の冒頭30文字)
- カードクリック → `AR-2026/[顧客フォルダ]/[案件キー]/サマリー.html` を別タブで開く
  - 顧客フォルダのマッピング: 玉寿会→玉寿会_さくら苑、信用産業→信用産業、松村→愛の治療院松村、等
  - リンク先が存在しない場合のフォールバックも考慮

### Section 4: 案件一覧テーブル(フィルタ・ソート対応)
列構成:
| 列 | 内容 |
|---|---|
| 案件キー | クリックでサマリーへ |
| 顧客略 | T-XXXリンク |
| 正式工事名 | |
| ステータス | 色付きバッジ |
| 最新見積額 | フォーマット表示 |
| 請求済み金額 | フォーマット表示 |
| 入金状況 | アイコン+色 |
| 施工場所 | |
| 旧案件番号 | |

機能:
- 列ヘッダクリックでソート(昇順/降順切替)
- 上部にフィルタバー:
  - 顧客略でフィルタ(プルダウン)
  - ステータスでフィルタ(プルダウン)
  - フリーテキスト検索(案件キー・正式工事名・備考を対象)
- 行数表示(全X件中Y件表示)

### Section 5: リマインドエリア
data.jsonから自動検出する条件:

| 条件 | 検出ロジック |
|---|---|
| 🔴 完工後30日以上請求未確定 | ステータスが「完了/施工中/施工予定でない」かつ「請求」配列が空 |
| 🔴 請求書発行後60日経過・未入金 | 請求.発行日から60日経過 && 入金状況=minyukin |
| 🟡 見積提出後30日経過・進展なし | 見積有り && 請求なし && ステータス=見積提出済み && 見積.発行日から30日経過 |
| 🟢 今週の施工予定 | 備考に日付パターン("X/Y施工"等)+今週内 |

各リマインドに案件キー・該当書類・経過日数を表示。

### Section 6: 売上推移グラフ
Chart.jsで月別棒グラフ:
- X軸: 過去6ヶ月+当月+将来3ヶ月
- Y軸: 金額(千円単位)
- 系列:
  - 確定請求(請求済みの金額)
  - 入金済み(請求のうち入金確認済み)
  - 見積中(請求未発生の見積)

### Section 7: タイムライン(ガントチャート風)
- 期間: 今日-30日 ~ 今日+90日
- 横軸: 日付
- 縦軸: 案件キー
- 表示: 案件の見積発行日 → 施工日 → 請求発行日 → 入金日のバー
- SVG実装(mermaidは複雑な日付軸が苦手なため不採用)

### Section 8: 取引先パネル
20件の取引先をカード表示:
- カード内容: ID・略称・正式名称・種別タグ・関連案件数
- ステータス: 連絡先補完待ち(touroku=空 or contact.tel=空)を黄色マーキング
- クリックで詳細モーダル展開(memo・連絡先・関連案件一覧表示)

### Section 9: 横断検索バー(ヘッダー直下に固定)
Lunr.js使用:
- 検索対象: 案件全フィールド+取引先全フィールド+備考
- 結果表示: 該当エリアまでスクロール+ハイライト

### Section 10: フッター
- アート・レイズ著作表記
- 各種ルール文書へのリンク(GitHub上のINDEX.md等)
- 最終更新日

## 🚨 落とし穴と対策

### 1. 顧客フォルダ名の不統一
案件キーの顧客略(玉寿会・松村・信用産業等)と、AR-2026配下のフォルダ名(玉寿会_さくら苑・愛の治療院松村・信用産業)が一致しない。

**対策**: 案件キー → フォルダ名のマッピングテーブルをdashboard.html内に定義
```javascript
const FOLDER_MAP = {
  "玉寿会": "玉寿会_さくら苑",
  "松村": "愛の治療院松村",
  "信用産業": "信用産業",
  "ウインズ": "ウインズコーポレーション",
  // ...
};
```

### 2. 金額の表示フォーマット
全金額に「¥」プレフィックス+3桁カンマ区切り+「税込」サフィックスを徹底。
```javascript
function formatYen(amount) {
  return `¥${amount.toLocaleString()}税込`;
}
```

### 3. 日付パース
data.jsonの日付はすべて`YYYY-MM-DD`形式。
空文字列の処理を忘れずに。

### 4. 案件サマリーHTML不存在問題
全案件にサマリーHTMLがあるとは限らない。
リンクは作るが、クリック時に404を考慮(target="_blank"でブラウザに任せる)。

### 5. data.json更新タイミング
GitHub Pagesは反映に1-2分かかる。
開発中はローカルファイルでテスト、デプロイ後にPages版で最終確認。

### 6. Chart.js初期化タイミング
DOMContentLoaded後にChart.js初期化を行う。
data.json fetch完了を待つこと。

### 7. Lunr.jsインデックス構築
data.json読込→Lunrインデックス構築→検索可能、の順序を守る。
インデックス構築中の検索は無効化する。
