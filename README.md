# 現場ナビPRO

株式会社アート・レイズ（大分県別府市）の建設・内装リフォーム業向け業務管理Webアプリ。
案件・取引先・原価・書類を一元管理するシングルページアプリケーション。

## 主要ファイル

| ファイル | 役割 |
|---|---|
| `index.html` | ダッシュボード本体（案件一覧・取引先・書類リンク） |
| `data.json` | 全データのソース（案件・取引先マスター・単価マスタ等） |
| `zumen_app.html` | 図面管理アプリ |
| `floor_plan.html` | 平面図エディタ（ワイヤーフレーム自動生成） |
| `材料割付_app_.html` | 床材・壁材の材料割付計算ツール |
| `部材台帳_アートレイズ.html` | 案件で使った部材・商品の台帳（写真添付・案件/商品逆引き・原価台帳から取込） |
| `parts_photos/` | 部材台帳でアップロードした商品写真の保存先 |
| `工具台帳_アートレイズ.html` | 工具・機材・車両の台帳（写真からAI自動入力・型番証跡・取扱説明書保管・一括棚卸し・メンテ履歴・現場持込チェックリスト） |
| `tool_photos/` | 工具台帳でアップロードした工具写真の保存先 |
| `tool_manuals/` | 工具台帳でアップロードした取扱説明書（PDF/画像）の保存先 |
| `工具台帳登録_SKILL_v20260611.md` | 工具台帳に商品を登録する手順スキル（取扱説明書・商品画像を一緒に保管） |
| `mitsumori_preview.html` | 見積書プレビュー・PDF出力 |

## data.json の構造

トップレベルキー：
- `最終更新`：YYYY-MM-DD
- `命名ルール`：案件キー・ファイル名・書類番号の規則
- `取引先マスター`：顧客・業者の一覧（T-001〜）
- `案件`：進行中・完了案件
- `中止案件`：参考保管
- `部材`：各案件で使った部材・商品の記録（部材台帳_アートレイズ.html用）
  - 各要素：`id` / `案件キー` / `商品名` / `メーカー` / `品番` / `カテゴリ` / `使用場所` / `数量` / `単位` / `備考` / `写真`(parts_photos/配下のパス配列) / `登録日`
  - 取込元の原価行は備考に `原価RC-26-xxx` を記録（部材台帳の「原価台帳から取込」で重複検知に使用）
  - 「原価台帳から取込」：原価管理スプレッドシートの行を貼り付けると、案件番号→案件キー（旧案件番号で解決）で紐付け、材料費・消耗品費を自動抽出して部材化。交通費・飲食・取込済み行は自動除外
- `工具`：保有する工具・機材・車両（工具台帳_アートレイズ.html用）
  - 各要素：`id` / `名称` / `カテゴリ` / `メーカー` / `型番` / `保有数` / `状態` / `保管場所` / `取得日` / `取得価格` / `仕入先` / `使用方法` / `応用` / `注意事項` / `メンテ履歴`(配列) / `次回点検` / `写真`(tool_photos/配下) / `銘板写真`(tool_photos/配下・型番証跡) / `取扱説明書`(tool_manuals/配下・`[{名前, パス}]`の配列) / `備考` / `原価ID`
  - 初期データは原価台帳の工具購入（RC-26-xxx）から作成
  - 登録時「写真からAIで自動入力」：工具写真をClaude(vision)に送り、名称/メーカー/型番/カテゴリ/使用方法/応用/注意事項を推定して各欄へ反映。APIキーは `localStorage('ANTHROPIC_API_KEY')` をai_assist.htmlと共用
  - 「銘板・型番ラベルで精度UP」：銘板/型番ラベルの接写を追加すると、全体写真と合わせて最大6枚を解析し、ラベルの印字（メーカー/型番/定格）を最優先で読み取り精度を上げる
  - 型番証跡：銘板写真は `銘板写真`(配列) として通常写真と別枠で保存（詳細画面に「型番証跡」として表示）
  - 取扱説明書：登録/編集フォームの「取扱説明書（PDF・画像・複数可）」欄からアップロード。`tool_manuals/<工具id>_<timestamp>.<拡張子>` に個別コミットし、`取扱説明書`(`[{名前, パス}]`)に記録。詳細画面にリンク表示。登録時は商品画像(`写真`)と取説をセットで保管する運用（工具台帳登録スキル参照）
  - 一括棚卸し：複数の工具写真を連続で追加→AIが各1点を判定→一覧で確認・修正して一括登録（`ソース`=一括棚卸し）
- `持込リスト`：案件ごとの現場持込工具チェックリスト（工具台帳用）
  - 各要素：`案件キー` / `items`([{`id`(工具id), `持出`, `返却`, `返却日`}]) / `更新日`
- `単価マスタ`：天井・壁・床の材料単価（材料割付_app_/floor_plan用）

## 開発

ローカル編集 → `auto_push.bat` でGitHubに自動push → GitHub Pagesに数分で反映。
data.jsonの単価マスタを書き換えると、現場ナビPRO側のヘッダーバッジ「🔗 単価マスタ連携中」経由で
最大6時間以内（バッジタップで即時）にアプリへ反映される。
