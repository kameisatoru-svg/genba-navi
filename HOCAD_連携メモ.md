## ■ HO_CAD pao について

岡崎宏之氏が開発したWindows用2次元汎用CADソフト。
DOS版JW-CADとコマンド操作の互換性が高い。MDI対応で複数図面を同時に開ける。

### 主要機能サマリー

| カテゴリ | 主な機能 |
|----------|----------|
| 作図 | 線引き・複線・線伸縮・コーナー連結・円弧・変形（パラメトリック・包絡） |
| 装飾・塗り | ペイント・ハッチ（斜線・シンボル） |
| 文字 | 引き出し線・バルーン・装飾文字・自動インクリメント・Excel連携テーブル |
| 寸法 | 縦横・任意方向・角度・半径・直径・円周・面積計測・求積表 |
| レイヤ・グループ | レイヤ（表示/非表示）・グループ（縮尺設定可） |
| 画像 | JPG貼付・縮尺合わせ・射影変換・回転・輝度調整 |
| 図形 | 図形登録・読込（部品再利用）・断面図作成 |
| 測量 | SIMA/CSV/WPT読込・Googleマップ連携・逆トラ・放射トラバース |
| 移動・複写 | 回転・縮尺違い図面の合成・割付図作成 |
| 入出力 | 印刷・SXF（P21/SFC）・DWG/DXF読込 |

### 設定・データ管理
- 設定ファイル：`HO_CADpao.ini`（環境ファイル書出/読込でPC移行可能）
- HOC変形データは初期インストールでは空 → 別途データ追加必要
- 建具は線記号変形で線上スライド配置だが**間口自由伸縮は非対応**
  （横方向のみ伸縮データが存在しない）→ よく使うサイズを図形登録するのが現実的

### 電気設備CADデータ入手先
- CAD素材.com
- architerial.jp
- CAD Japan.com

## ■ A3図面フォーマット構造（9マス）

A3用紙1枚に9枚の図面枠を3列×3行で配置。
各枠が「親レイヤー（図面種別）」に対応する。

### 親レイヤー一覧

| 番号 | 種別 | 配置位置 |
|------|------|----------|
| 0 | 平面図 | 中段中央（メイン） |
| 1 | 展開図1 | 上段左 |
| 2 | 展開図2 | 上段中 |
| 3 | 展開図3 | 上段右 |
| 4 | 展開図4 | 中段左 |
| 5 | 立面図1 | 中段右 |
| 6 | 平面図2 | 下段左 |
| 7 | 家具図 | 下段中 |
| 8 | その他 | 下段右 |

### 子レイヤー（各親レイヤーの下に属する描画レイヤー）
- 現在DXFで確認されているもの：`躯体`・`照明`・`G_3-L_0`〜`G_8-L_0`・`ﾌﾚｰﾑ`
- 各親への割り当て詳細は未説明 → **次回以降追記予定**

### レイヤー分け推奨（Claude理解精度向上のため）

現状は `躯体`・`照明` の2レイヤーにオールイン → Claudeが壁と家具を区別できない。

**最低限この3つで効果大**
```
壁 / 家具 / 寸法
```

**理想の7レイヤー**
```
壁 / 建具 / 家具 / 寸法 / 照明 / 電気 / 注記
```

日本語レイヤー名はDXF書き出し時そのまま出力され、Claudeは完全読み取り可能。
メインレイヤー・サブレイヤーの階層構造もDXF出力後にClaudeが完全理解可能。

## ■ Claude連携ワークフロー

### DXF解析 Claudeの能力と制約

| 条件 | Claudeの対応 |
|------|--------------|
| ベクター線（壁・寸法・家具等） | ✅ 座標・寸法の数値解析が可能 |
| JPG画像埋め込み | ❌ ベクター解析不可（画像として認識のみ） |
| レイヤー名・構造 | ✅ 読み取り可能 |
| テキスト情報 | ✅ Shift-JIS変換で読み取り可能 |
| DXF出力（Claude→HO_CAD） | ❌ HO_CADクラッシュのため断念済み |

**推奨：図面はベクター線で描き、画像貼り込みは最小限に。**

### 標準ワークフロー（DXF＋PDF同時渡しが最強）

```
①亀井さんがHO_CADで図面作成（ベクター線）
　　↓
②DXF書き出し（Shift-JIS・HO_CAD pao形式）＋PDF同時出力
　　↓
③ClaudeがDXFを解析
　　・DXF：座標・寸法・設備記号の数値情報（図形的なもの）
　　・PDF：空間構成・配置意図の視覚的確認（感性的なもの）
　　↓
④Claude復唱 → 亀井さん正誤確認
　　↓
⑤完全理解後 → 透視図・パース生成・見積補助等
```

### Geminiパース生成ワークフロー

- **プロンプトで指示**：質感・色・雰囲気・素材（感性的なもの）
- **DXF/PDFで伝える**：空間形状・家具配置・寸法（図形的なもの）
- ClaudeがmatplotlibでワイヤーフレームPNG/PDFを生成し、Geminiに骨格として渡す方法も有効
